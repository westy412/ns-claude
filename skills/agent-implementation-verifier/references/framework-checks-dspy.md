# DSPy Framework Compliance Checks

> **When to read:** When the agent-config.yaml specifies `framework: dspy`. Include this content in the framework-compliance-verifier agent's prompt.

---

## Two-File Prompt Pattern (Critical)

The DSPy prompt pattern uses `signatures.py` + `prompts/{agent_name}.md`. This is the most important framework-specific check.

| Check | Status if violated |
|-------|--------------------|
| `signatures.py` has empty docstrings (`""""""`) for each signature | FAIL |
| `_load_prompt()` helper function exists in signatures.py | FAIL |
| `__doc__` assignments exist at module level for each signature | FAIL |
| Corresponding `prompts/{agent_name}.md` file exists for each signature | FAIL |
| Prompt .md files use XML tags for structure (not `=== SECTION ===` headers) | FAIL |
| Prompts are substantive (20+ lines, not brief stubs) | WARN |
| NO `prompts.py` file exists (this is the LangGraph pattern, not DSPy) | FAIL |

---

## Required Files

| Check | Status if violated |
|-------|--------------------|
| `utils.py` EXISTS (it is required in DSPy projects, not optional) | FAIL |
| `utils.py` contains singleton LM factory functions | FAIL |
| Orchestration file is named `team.py` (not `program.py` or `pipeline.py`) | FAIL |

---

## Module Type Compliance

| Agent Type in Spec | Expected DSPy Module | Check |
|-------------------|---------------------|-------|
| `basic-agent` | `dspy.Predict` | Uses Predict, not ChainOfThought |
| `reasoning-agent` | `dspy.ChainOfThought` | Uses ChainOfThought |
| `conversational-agent` | `dspy.Predict` with conversation history | Has conversation context in signature |
| `tool-agent` | `dspy.ChainOfThought` + `dspy.ToolCalls` (default) OR `dspy.ReAct` (rare — only for multi-step dynamic tool chains) | Default: uses ChainOfThought/Predict with manual tool execution. ReAct only if spec explicitly justifies multi-step dynamic tool selection. |

Flag module type mismatches as FAIL. Note: `tool-agent` using ChainOfThought + ToolCalls is the **preferred** pattern. Only flag as FAIL if the agent uses neither ToolCalls nor ReAct.

## Async Pattern Compliance

| Check | Status if violated |
|-------|--------------------|
| Every `dspy.Module` has both `forward()` and `aforward()` methods | FAIL |
| `aforward()` uses `await agent.acall()` (not `agent()`) | FAIL |
| `aforward()` uses `call_with_retry` wrapper | WARN |
| `team.py` has both `forward()` and `aforward()` | FAIL |

---

## Singleton LM Pattern

| Check | Status if violated |
|-------|--------------------|
| LM instances are created via factory functions (e.g., `get_flash_lm()`, `get_pro_lm()`) | FAIL |
| `dspy.LM()` is NOT called directly in agent code (only in utils.py factories) | FAIL |
| Model tier assignments match spec (Flash for simple agents, Pro for complex) | WARN |

---

## Directory and Naming Conventions

| Check | Status if violated |
|-------|--------------------|
| Directory names use `snake_case` (Python package requirement, not kebab-case) | FAIL |
| Sub-teams sit directly under `src/`, not inside wrapper directories like `src/programs/` | FAIL |
| No `src/routes/` directory exists (endpoints go in `main.py`) | FAIL |
| Prompt files in `prompts/` use `{agent_name}.md` naming | WARN |

---

## I/O and Typing

| Check | Status if violated |
|-------|--------------------|
| Typed outputs used (no `str` + JSON parsing patterns) | FAIL |
| All spec inputs have corresponding `InputField` or function parameter | FAIL |
| All spec outputs have corresponding `OutputField` or return type | FAIL |
| `code-resolved` fields handled in post-processing, not left to LLM | FAIL |
| `pass-through` fields copied correctly (not regenerated by LLM) | WARN |
| Tool functions return `dict` (not Pydantic models or raw values) | FAIL |

---

## Multi-Team / Nested Team

| Check | Status if violated |
|-------|--------------------|
| Each sub-team has its own `team.py`, `signatures.py`, `prompts/`, `utils.py` | FAIL |
| Signatures and prompts are NOT shared between sibling teams | FAIL |
| Shared infrastructure lives at parent level | WARN |
| Sub-teams are standalone modules (no factory function generation) | FAIL |
| For multi-team services: request schemas use inheritance (CommonContext base) | WARN |

---

## Anti-Pattern Summary

| Anti-Pattern | Detection | Severity |
|-------------|-----------|----------|
| `prompts.py` file exists (LangGraph pattern in DSPy project) | File exists check | FAIL |
| `dspy.LM()` called directly in agent code | Grep for `dspy.LM(` outside utils.py | FAIL |
| Missing `utils.py` | File existence check | FAIL |
| Missing `aforward()` on any Module | Grep for class definitions, check for both methods | FAIL |
| Non-empty signature docstrings | Grep for docstrings in signatures.py that aren't `""""""` | FAIL |
| `program.py` or `pipeline.py` used instead of `team.py` | File existence check | FAIL |
| Wrapper directory like `src/programs/` | Directory structure check | FAIL |
| String outputs parsed as JSON | Grep for `json.loads` on agent outputs | FAIL |
